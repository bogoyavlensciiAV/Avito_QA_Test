# API Test Cases

## Общая информация
**Базовый URL:** `https://qa-internship.avito.com`

**Ручки микросервиса:**
1. `POST /api/1/item` — Создать объявление
2. `GET /api/1/item/:id` — Получить объявление по идентификатору
3. `GET /api/1/:sellerID/item` — Получить все объявления по идентификатору продавца
4. `GET /api/1/statistic/:id` — Получить статистику по идентификатору объявления

---

## Модульные тест-кейсы API

### 1. `POST /api/1/item` (Создать объявление)

| ID | Тип | Описание | Действие | Ожидаемый результат |
|:---|:---|:---|:---|:---|
| 1.1 | Позитивный | Создание с валидными положительными данными | Отправить POST-запрос с `sellerID=111111`, `price=100`, `likes=10`, `viewCount=50`, `contacts=3`. | Статус-код 200 OK. В ответе сгенерирован уникальный `id`. |
| 1.2 | Позитивный | Создание с нулевыми значениями цены и статистики | Отправить POST-запрос с `price=0`, `likes=0`, `viewCount=0`, `contacts=0`. | Статус-код 200 OK. Объявление создается с нулевыми значениями. |
| 1.3 | Негативный | В качестве цены и статистики переданы отрицательные числа | Отправить POST-запрос с `price=-100`, `likes=-1`, `viewCount=-5`, `contacts=-1`. | Статус-код 400 Bad Request. |
| 1.4 | Негативный | Создание объявления без обязательного поля `sellerID` | Отправить POST-запрос без поля `sellerID`. | Статус-код 400 Bad Request. |
| 1.5 | Негативный | Создание объявления с неверным типом данных | Отправить POST-запрос со строковым значением `price="сто"`. | Статус-код 400 Bad Request. |
| 1.6 | Корнер-кейс | Дублирование идентичных запросов | Отправить два одинаковых POST-запроса подряд. | Оба возвращают 200 OK. Генерируются два разных уникальных `id`. |

### 2. `GET /api/1/item/:id` (Получить объявление по id)

| ID | Тип | Описание | Действие | Ожидаемый результат |
|:---|:---|:---|:---|:---|
| 2.1 | Позитивный | Получение существующего объявления | Отправить GET-запрос с `id`, который существует в базе. | Статус-код 200 OK. Возвращаются данные объявления, совпадающие с созданным объектом. |
| 2.2 | Негативный | Получение объявления по несуществующему `id` | Отправить GET-запрос с несуществующим `id` (например, случайный UUID). | Статус-код 404 Not Found. |
| 2.3 | Негативный | Получение объявления с невалидным форматом `id` | Отправить GET с некорректным форматом `id` (спецсимволы). | Статус-код 400 Bad Request или 404 Not Found. |

### 3. `GET /api/1/:sellerID/item` (Получить объявления продавца)

| ID | Тип | Описание | Действие | Ожидаемый результат |
|:---|:---|:---|:---|:---|
| 3.1 | Позитивный | Получение объявлений продавца | Отправить GET-запрос с `sellerID`, у которого есть сохраненные объявления. | Статус-код 200 OK. В ответе возвращается массив объявлений данного продавца. |
| 3.2 | Корнер-кейс | Запрос для `sellerID`, у которого нет объявлений | Отправить GET-запрос с `sellerID`, на который не создавалось объявлений. | Статус-код 200 OK и пустой массив `[]` ИЛИ 404 Not Found. |
| 3.3 | Негативный | Вызов с невалидным `sellerID` | Отправить GET-запрос с текстовым `sellerID="abc"`. | Статус-код 400 Bad Request. |
| 3.4 | Негативный | Вызов с отрицательным `sellerID` | Отправить GET-запрос с `sellerID=-111111`. | Статус-код 400 Bad Request. |

### 4. `GET /api/1/statistic/:id` (Получить статистику)

| ID | Тип | Описание | Действие | Ожидаемый результат |
|:---|:---|:---|:---|:---|
| 4.1 | Позитивный | Статистика существующего объявления | Запрос статистики по существующему `id`. | Статус-код 200 OK. Возвращается статистика, соответствующая объявлению. |
| 4.2 | Негативный | Статистика по несуществующему `id` | Запрос статистики по несуществующему `id`. | Статус-код 404 Not Found. |

## E2E сценарии 

### E2E-1. Полный цикл: создание и получение объявления

| Шаг | Начальное состояние | Действие | Ожидаемый результат | Конечное состояние |
|:---|:---|:---|:---|:---|
| 1 | Объявления не существует | Вызвать `POST /api/1/item` (sellerID=111111, name="Тест", price=100) | Возвращается 200 OK и сгенерированный `id` | Объявление существует |
| 2 | Объявление существует | Вызвать `GET /api/1/item/:id` с полученным ID | Возвращается 200 OK. Данные полностью совпадают с созданным | Объявление получено |

### E2E-2. Полный цикл: создание и проверка статистики

| Шаг | Начальное состояние | Действие | Ожидаемый результат | Конечное состояние |
|:---|:---|:---|:---|:---|
| 1 | Объявления не существует | Вызвать `POST /api/1/item` (передать statistics: likes=10, viewCount=5, contacts=3) | Возвращается 200 OK и сгенерированный `id` | Объявление существует вместе со статистикой |
| 2 | Статистика существует | Вызвать `GET /api/1/statistic/:id` с полученным ID | Возвращается 200 OK. Массив статистики совпадает с переданным при публикации | Статистика получена |

### E2E-3. Полный цикл: публикация нескольких объявлений и получение списка продавца

| Шаг | Начальное состояние | Действие | Ожидаемый результат | Конечное состояние |
|:---|:---|:---|:---|:---|
| 1 | У продавца 123456 нет объявлений | Вызвать `POST /api/1/item` для публикации объявления с `sellerID`=123456 | Возвращается 200 OK | У продавца 1 объявление |
| 2 | У продавца 1 объявление | Вызвать снова `POST /api/1/item` (другое объявление) с `sellerID`=123456 | Возвращается 200 OK | У продавца 2 объявления |
| 3 | У продавца 2 объявления | Вызвать `GET /api/1/123456/item` | Возвращается 200 OK и массив из двух элементов | Объявления продавца выгружены и получены |
